# 潮汐图床存储适配器设计

## 1. 设计目标

潮汐图床的存储层需要统一接管：

- 原图上传。
- WebP 预览图上传。
- 头像上传。
- 图片读取。
- 图片删除。
- 图片访问 URL 生成。
- 随机图调用 URL 返回。
- 图片迁移。
- 健康检查。

业务层不直接拼接存储 URL，也不直接调用某一种云存储。

## 2. 统一接口

所有存储后端实现统一 StorageProvider：

```text
put_object
get_object
delete_object
head_object
get_public_url
create_presigned_url
health_check
refresh_auth
```

当前实现：

- LocalStorageProvider。
- S3CompatibleProvider，用于 Cloudflare R2、Oracle S3 和其他 S3 兼容服务。
- OneDriveProvider。
- OracleOciNativeProvider。

## 3. 对象路径规范

原图：

```text
images/{yyyy}/{mm}/{dd}/{sha256}.{ext}
```

预览图：

```text
previews/{yyyy}/{mm}/{dd}/{sha256}.webp
```

头像：

```text
avatars/{user_id}/{sha256}.webp
```

备份：

```text
backups/{yyyy}/{mm}/{dd}/{backup_id}.tar.zst
```

## 4. 存储策略

后台可配置两层策略：

1. 存储 provider：定义磁盘或云存储本身，包括类型、根目录、公开前缀、公开域名、路径前缀、凭据、启用状态和全局默认标记。
2. 存储路由：定义上传时不同人群应该写入哪个 provider。

路由作用域：

- `user`：指定用户 UUID。
- `group`：指定用户组 code。
- `role`：指定用户角色。
- `global`：全局兜底。

上传选择顺序：

```text
用户路由
↓
用户组路由
↓
角色路由
↓
全局路由
↓
用户组配额 default_storage_provider_id
↓
全局默认 provider
```

同一作用域有多条路由时，`priority` 数字越小越优先。路由只会命中启用状态的路由和启用状态的 provider。

推荐实践：

- 本地或低成本 S3 兼容存储承接普通用户上传。
- R2 或公开域名稳定的 S3 provider 承接可信用户和公开图库。
- OneDrive 或 OCI Object Storage 承接归档、迁移和备份。
- 至少保留一条启用的全局默认 provider，避免路由误配置导致上传不可用。

## 5. 上传流程

```text
用户上传图片
↓
后端计算 SHA-256
↓
检查 file_objects 是否已存在
↓
如果不存在，生成原图 object_key
↓
调用 StorageProvider.put_object 上传原图
↓
生成 WebP 预览图
↓
调用 StorageProvider.put_object 上传预览图
↓
写入 storage_objects
↓
写入 images 用户引用记录
```

## 6. 访问流程

图片详情、Markdown、HTML、随机图统一通过存储器生成最终访问地址。

根据存储模式返回：

- 公开存储：`public_url`。
- 私有存储：presigned_url。
- OneDrive：默认系统代理 URL。
- 审核中/私有图：系统代理 URL。

前后端链接模式：

- 前端 API 请求使用同源相对路径，例如 `/api/images/upload`。
- 后端按 `PUBLIC_BASE_URL` 归一化本地公开链接。
- 前端复制链接时会把同源 `/files/` 和 `/api/storage/proxy/` 转为当前浏览器 origin 下的完整 URL。
- 本地开发、Docker 直连、反向代理和公网域名部署都使用同一套链接生成逻辑。

## 7. Cloudflare R2

接入方式：

- S3 兼容 API。

后台配置：

- 存储名称。
- Account ID。
- Access Key ID。
- Secret Access Key。
- Bucket。
- Jurisdiction：Global、EU、FedRAMP。
- Endpoint 覆盖，可选；为空时由 Account ID 和 Jurisdiction 自动生成。
- Region 覆盖，可选；默认 `auto`。
- Public Domain。
- Path Prefix。
- 访问模式。
- Presigned URL TTL。
- 是否启用。
- 是否默认。

Endpoint 生成：

- Global：`https://{account_id}.r2.cloudflarestorage.com`。
- EU：`https://{account_id}.eu.r2.cloudflarestorage.com`。
- FedRAMP：`https://{account_id}.fedramp.r2.cloudflarestorage.com`。

访问模式：

- 公开域名直出。
- 签名 URL。
- 系统代理。

图床场景推荐：

- 公开图片：自定义域名直出。
- 私有图片：签名 URL 或系统代理。
- 预览图：公开直出。

## 8. OneDrive

### 8.1 接入方式

OneDrive 使用 Azure App 信息接入。

管理员填写：

- 客户端 ID。
- 租户 ID。
- 客户端密钥。
- 邮箱。
- 根目录。
- 可选 refresh token。

适用：

- Microsoft 365 E3。
- OneDrive for Business。
- 企业/组织租户。
- 使用 refresh token 时也可覆盖个人 OneDrive 或限制应用权限访问 `/users/{email}/drive` 的租户。

### 8.2 认证流程

默认使用 Microsoft Graph Client Credentials 流程：

```text
管理员填写 client_id、tenant_id、client_secret、email
↓
Rust 后端请求 Microsoft Token
↓
通过邮箱查找用户 Drive
↓
获取 drive_id
↓
测试创建目录和上传文件
↓
保存配置
```

Token Scope：

```text
https://graph.microsoft.com/.default
```

当配置 `refresh_token` 时使用 delegated token 刷新流程：

```text
管理员填写 client_id、tenant_id、client_secret、refresh_token、root_dir
↓
Rust 后端刷新 Microsoft Token
↓
通过 /me/drive 访问当前授权账号 Drive
↓
测试上传、读取和删除文件
```

### 8.3 权限要求

Azure App 需要管理员同意：

- Files.ReadWrite.All。
- User.Read.All。

如果连接或上传测试返回 401/403，后端会将 Microsoft Graph 的 `code/message` 转为可读错误。常见原因包括 client secret 过期、tenant/client 不匹配、应用权限未管理员同意、目标用户 OneDrive 未初始化，或租户禁止应用权限访问目标用户 Drive；后一类场景可改用 delegated `refresh_token` 模式。

### 8.4 上传方式

小文件：

```text
PUT /users/{email}/drive/root:/path/to/file:/content
```

大文件：

```text
POST /users/{email}/drive/root:/path/to/file:/createUploadSession
```

### 8.5 访问模式

- 系统代理访问，推荐默认。
- 分享链接访问，可选。

推荐系统代理原因：

- 更好控制私有图片。
- 更好支持审核和回收站。
- 不依赖 OneDrive 分享链接长期稳定性。

## 9. Oracle Object Storage

### 9.1 支持模式

第一版同时支持：

- S3 兼容模式。
- OCI 原生模式。

### 9.2 Oracle S3 兼容模式

配置项：

- Endpoint。
- Region。
- Namespace。
- Bucket。
- Access Key ID。
- Secret Access Key。
- Public Domain。
- Path Prefix。
- 访问模式。

Endpoint 示例：

```text
https://{namespace}.compat.objectstorage.{region}.oraclecloud.com
```

### 9.3 Oracle OCI 原生模式

配置项：

- Region。
- Namespace。
- Bucket。
- Compartment OCID。
- Tenancy OCID。
- User OCID。
- Fingerprint。
- Private Key。
- Private Key Passphrase，可选。
- Public Domain，可选。
- Path Prefix。
- 访问模式。

### 9.4 OCI 原生功能

支持：

- 上传原图。
- 上传 WebP 预览图。
- 上传头像。
- 删除对象。
- 读取对象元数据。
- 生成公开访问地址。
- 生成 PAR 预认证链接。
- 系统代理访问。
- 健康检查。
- 测试上传。
- 测试删除。

### 9.5 OCI 访问模式

- 公开 Bucket URL。
- PAR 预认证访问。
- 签名访问。
- 系统代理。

## 10. 本地存储

配置项：

- 本地根目录。
- 公开访问 URL 前缀。
- Path Prefix。
- 是否启用。
- 是否默认。

本地存储主要用于：

- 开发环境。
- 小规模部署。
- 临时缓存。
- 迁移中转。

## 11. 图片迁移

支持任意存储到任意存储：

- 本地到 R2。
- 本地到 OneDrive。
- 本地到 Oracle。
- R2 到 OneDrive。
- R2 到 Oracle。
- OneDrive 到 R2。
- Oracle 到 R2。

迁移模式：

- 复制模式。
- 移动模式。
- 备份模式。

迁移要求：

- 断点续传。
- 失败重试。
- 迁移日志。
- 迁移进度。
- 并发数量限制。
- 失败文件列表。
- 完成后校验哈希。
