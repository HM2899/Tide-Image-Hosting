# 潮汐图床数据库设计

## 1. 数据库

主数据库使用 PostgreSQL。

设计原则：

- 真实文件和用户图片引用分离。
- 存储位置和逻辑文件分离。
- 敏感配置加密保存。
- 删除优先软删除。
- 审核、迁移、备份都使用任务表记录状态。

## 13. storage_routes

上传存储路由表。用于把不同用户、用户组、角色或全局上传流量路由到指定存储 provider。

```text
id
name
scope_type
scope_value
storage_provider_id
enabled
priority
note
created_at
updated_at
```

scope_type：

```text
global
role
group
user
```

说明：

- `global` 的 `scope_value` 为空。
- `role` 的 `scope_value` 为用户角色，例如 `user`、`trusted`、`supporter`、`admin`。
- `group` 的 `scope_value` 为用户组 code。
- `user` 的 `scope_value` 为用户 UUID。
- 上传匹配顺序为用户、用户组、角色、全局、配额默认存储、全局默认 provider。
- `priority` 数字越小越靠前，同一作用域命中多条时优先级更小者生效。
