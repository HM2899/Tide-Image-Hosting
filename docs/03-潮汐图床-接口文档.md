# 潮汐图床接口文档

## 1. 约定

接口前缀：

```text
/api
```

认证方式：

- Web 登录：Session。
- API 调用：Bearer Token。
- 管理员接口：需要管理员权限。

通用响应：

```json
{
  "success": true,
  "data": {},
  "message": ""
}
```

错误响应：

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误说明"
  }
}
```

## 4. 图片上传接口

登录用户上传：

```text
POST /api/images/upload
```

访客上传：

```text
POST /api/guest/images/upload
```

上传返回：

```json
{
  "id": "image_id",
  "url": "https://example.com/image.jpg",
  "preview_url": "https://example.com/preview.webp",
  "markdown": "![image](https://example.com/image.jpg)",
  "html": "<img src=\"https://example.com/image.jpg\" alt=\"image\">",
  "preview_markdown": "![image](https://example.com/preview.webp)",
  "preview_html": "<img src=\"https://example.com/preview.webp\" alt=\"image\">",
  "status": "active",
  "deduplicated": false
}
```

字段约定：

- `url`、`markdown`、`html` 始终引用原图。
- `preview_url`、`preview_markdown`、`preview_html` 始终引用 WebP 预览图；历史图片没有 WebP 时可回退到原图或返回空值。
- 服务端返回本地存储历史绝对地址时，必须按当前 `PUBLIC_BASE_URL` 归一化同站 `/files/` 和 `/api/storage/proxy/` 链接。
- 上传接口成功后先返回 `active` 以便正常展示，审核任务异步执行；审核拒绝后图片会被永久删除并保留审核记录。

## 5.1 公开图库接口

```text
GET /api/public/images
```

公开图库接口无需登录，只返回 `status=active` 且 `visibility=public` 的图片。该接口不得返回隐藏、私有、待审核、回收站、已删除、已隔离图片。

查询参数：

```text
tag
orientation
min_width
min_height
page
page_size
```

返回：

```json
{
  "success": true,
  "data": {
    "items": [],
    "page": 1,
    "page_size": 60,
    "total": 0
  },
  "message": ""
}
```

## 12. 存储管理接口

后台前端将存储 provider、存储路由、健康检查和图片迁移归入“存储与迁移”一级菜单。接口路径保持不变。

```text
GET    /api/admin/storage/providers
POST   /api/admin/storage/providers
GET    /api/admin/storage/providers/{provider_id}
PUT    /api/admin/storage/providers/{provider_id}
DELETE /api/admin/storage/providers/{provider_id}
POST   /api/admin/storage/providers/{provider_id}/test-connection
POST   /api/admin/storage/providers/{provider_id}/test-upload
POST   /api/admin/storage/providers/{provider_id}/test-delete
POST   /api/admin/storage/providers/{provider_id}/set-default
GET    /api/admin/storage/health
GET    /api/admin/storage/health/page?ids=
GET    /api/admin/storage/routes
GET    /api/admin/storage/routes/page?page=1&page_size=20&q=&role=&status=
GET    /api/admin/storage/routes/summary
POST   /api/admin/storage/routes
GET    /api/admin/storage/routes/{route_id}
PUT    /api/admin/storage/routes/{route_id}
POST   /api/admin/storage/routes/{route_id}/enable
POST   /api/admin/storage/routes/{route_id}/disable

```

`POST /api/admin/storage/providers`、`PUT /api/admin/storage/providers/{provider_id}` 和 `POST /api/admin/storage/providers/{provider_id}/set-default` 成功时返回脱敏后的 provider 对象，前端可直接局部更新当前列表。`test-connection`、`test-upload`、`test-delete` 失败时返回标准错误体；OneDrive 错误会包含 Microsoft Graph 的 `code/message` 和权限排查提示，避免只显示 502。

OneDrive provider 配置字段：

```json
{
  "client_id": "Azure App Client ID",
  "tenant_id": "Azure Tenant ID",
  "client_secret": "Azure Client Secret",
  "email": "drive-owner@example.com",
  "refresh_token": "可选，使用 delegated /me/drive 时填写",
  "root_dir": "TideImages",
  "path_prefix": "可选路径前缀"
}
```

`POST /api/admin/storage/routes` 请求体：

```json
{
  "name": "普通用户走本地磁盘 2",
  "scope_type": "role",
  "scope_value": "user",
  "storage_provider_id": "uuid",
  "enabled": true,
  "priority": 100,
  "note": "可选备注"
}
```

路由匹配顺序：

```text
user > group > role > global > quota_rule.default_storage_provider_id > default_provider
```

## 15.1 前端错误展示规则

前端不得直接展示完整 JSON 错误、HTML 错误页或原始响应体。所有接口错误必须转换为用户可读短提示：

- `UNAUTHORIZED`：请先登录。
- `FORBIDDEN`：没有权限或后台已关闭该能力。
- `FILE_TOO_LARGE`：文件超过大小限制。
- `FILE_TYPE_NOT_ALLOWED`：文件类型不允许。
- `QUOTA_EXCEEDED`：配额不足。
- `DATABASE_ERROR`：系统繁忙，请稍后重试。
