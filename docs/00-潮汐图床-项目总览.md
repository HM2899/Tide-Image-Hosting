# 潮汐图床项目总览

## 1. 项目定位

潮汐图床是一个以 Rust 为主技术栈开发的公益图床系统，目标是提供稳定、安全、省存储、可维护、可扩展的图片托管服务。

项目第一版不追求复杂社区化，优先完成图床核心闭环：

- 用户注册、登录、邮箱验证。
- 普通用户图片上传、查看、删除、恢复。
- 访客免登录上传，并归属到指定访客公用账号。
- 管理员后台。
- 多存储后端。
- 哈希去重。
- WebP 预览图。
- 用户组配额。
- 回收站。
- 图片标签。
- 随机图接口。
- 审核系统。
- 移动端 UI。
- 图片迁移。
- 网站备份。

## 2. 技术路线

主系统：

- 后端语言：Rust
- 后端框架：Axum
- 前端框架：Leptos
- 数据库：PostgreSQL
- ORM / SQL：SQLx
- 异步运行时：Tokio
- 缓存与限流：Redis，可选
- 图片处理：Rust image 生态相关库
- 认证：Session + JWT / API Token

AI 服务：

- 使用独立 FastAPI 微服务。
- Rust 主服务通过 HTTP 调用 FastAPI。
- FastAPI 负责 AI 审核、OCR、大模型调用、图片描述、图片标签建议等能力。

## 3. 第一版必须支持的存储

- 本地存储。
- Cloudflare R2。
- OneDrive。
- Oracle Object Storage S3 兼容模式。
- Oracle Object Storage OCI 原生模式。

后续可扩展：

- 通用 S3 Compatible。
- 阿里云 OSS。
- 腾讯云 COS。
- Backblaze B2。

## 4. 核心角色

- 访客用户：未登录用户，是否允许上传由后台开关控制。
- 普通用户：邮箱注册后上传和管理自己的图片。
- 可信用户：管理员手动提升，配额更高，审核更宽松。
- 公益支持者：面向贡献者、捐赠者或社区成员的高配额用户组。
- 管理员：通过普通登录入口登录，登录后显示管理员后台入口。
- 超级管理员：可修改关键系统配置和管理管理员。

## 5. 第一版后台一级菜单

- 仪表盘。
- 图片与标签。
- 用户与配额。
- 审核管理。
- 存储与迁移。
- 系统设置。

后台采用 6 个一级入口。上传、随机图、外观、日志和备份恢复收纳到系统设置；标签能力保留在图片与标签中，不再作为单独一级菜单。
